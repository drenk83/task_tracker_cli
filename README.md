# task_tracker_cli

Build a CLI app to track your tasks and manage your to-do list

Трекер задач — это проект для отслеживания и управления вашими задачами. В рамках этого задания вы создадите простой интерфейс командной строки (CLI), чтобы фиксировать, что вам нужно сделать, что уже выполнено и над чем вы сейчас работаете. Этот проект поможет вам попрактиковаться в программировании — в частности, в работе с файловой системой, обработке пользовательского ввода и создании простого CLI‑приложения.

### Требования

Приложение должно запускаться из командной строки, принимать действия и ввод пользователя в качестве аргументов и сохранять задачи в JSON‑файле. Пользователь должен иметь возможность:

* добавлять, обновлять и удалять задачи;
* помечать задачу как находящуюся в работе или выполненную;
* просматривать список всех задач;
* просматривать список всех выполненных задач;
* просматривать список всех невыполненных задач;
* просматривать список всех задач, находящихся в работе.

Вот некоторые ограничения, которыми следует руководствоваться при реализации:

* Вы можете использовать любой язык программирования для создания этого проекта.
* Используйте позиционные аргументы в командной строке для приёма ввода от пользователя.
* Для хранения задач используйте JSON‑файл в текущей директории.
* JSON‑файл должен создаваться, если он не существует.
* Для взаимодействия с JSON‑файлом используйте встроенный модуль работы с файловой системой вашего языка программирования.
* Не используйте какие‑либо внешние библиотеки или фреймворки для создания этого проекта.
* Обеспечьте корректную обработку ошибок и пограничных случаев.

### Пример

```bash
# Adding a new task
task-cli add "Buy groceries"
# Output: Task added successfully (ID: 1)

# Updating and deleting tasks
task-cli update 1 "Buy groceries and cook dinner"
task-cli delete 1

# Marking a task as in progress or done
task-cli mark-in-progress 1
task-cli mark-done 1

# Listing all tasks
task-cli list

# Listing tasks by status
task-cli list done
task-cli list todo
task-cli list in-progress

```

### Свойства задачи

Каждая задача должна иметь следующие свойства:

* id — уникальный идентификатор задачи;
* description — краткое описание задачи;
* status — статус задачи (todo, in‑progress, done);
* createdAt — дата и время создания задачи;
* updatedAt — дата и время последнего обновления задачи.

Обязательно добавляйте эти свойства в JSON‑файл при создании новой задачи и обновляйте их при изменении задачи.
